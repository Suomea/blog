一般情况下防火墙作用于 LAN 的边界，LAN 内部的主机为信任网域。

常用的防火墙软件 iptables，nftables 两个。

iptables 老牌的防火墙软件，基于 Netfilter 框架，使用范围较广，通过一系列独立的工具（iptables、ip6tables、ebtables、arptables）管理 IPv4、IPv6 等协议。

nftables 性能更高，语法简洁，功能更强大，扩展性更好，支持更多的新特性。

当网络流量进入防火墙的时候，防火墙会分析网络包然后按照顺序匹配预设的一系列规则，每个规则都会给出 ACCEPT 或 DROP。如果网络包能匹配到规则，则按照规则规定的动作执行，否则继续尝试匹配下一条规则。

一般防火墙会设置一个默认的规则，当所有的规则都没有匹配时，执行预设的规则。
## iptables
ipbtables 默认有三个表：

- 管理本机进出的 filter 表。
- 管理后端主机的 nat 表。
- 管理特殊旗标的 mangle 表。

查看规则，默认查看 filter 表：
```
iptables -L -n -v --line-number
```

删除规则：
```
iptables -D INPUT [num]
```

插入规则，插入到第五行（省略数字 5 的话，默认插入到第一行），只接受建立连接的 tcp 包且端口号为 18094：
```
iptables -I INPUT 5 -m state --state NEW  -p tcp  --dport 18094  -j ACCEPT
iptables -I INPUT 5 -m conntrack --ctstate NEW -p tcp --dport 18094 -j ACCEPT
```

使用 -A 为追加到最后一行，两个语句等效，下面一个更推荐：
```
iptables -A INPUT -m state --state NEW  -p tcp  --dport 18094  -j ACCEPT
iptables -A INPUT -m conntrack --ctstate NEW -p tcp --dport 18094 -j ACCEPT
```

实例分析
设置主机的 16000 端口只能本地、192.168.31.12、192.168.31.15-192.168.31.20 访问，其它访问全部拒绝
```
# 1. 设置已连接的不影响后续访问
iptables -I INPUT -m conntrack --ctstate RELATED,ESTABLISHED -j ACCEPT

# 2. 设置允许和拒绝的策略
iptables -A INPUT -p tcp -s 127.0.0.1 --dport 16001 -j ACCEPT
iptables -A INPUT -p tcp -s 192.168.31.12 --dport 16001 -j ACCEPT
iptables -A INPUT -p tcp -m iprange --src-range 192.168.31.15-192.168.31.20 --dport 16001 -j ACCEPT
iptables -A INPUT -p tcp --dport 16001 -j DROP
```

分析主机的策略
```
# 1. 允许已建立的连接和相关的流量（放行返回包，比如你访问网站后的响应）
iptables -A INPUT -m conntrack --ctstate RELATED,ESTABLISHED -j ACCEPT

# 或者旧版写法（如果 conntrack 不可用）：
iptables -A INPUT -m state --state RELATED,ESTABLISHED -j ACCEPT

# 2. 允许 ICMP (Ping) 请求
iptables -A INPUT -p icmp --icmp-type echo-request -j ACCEPT

# 或者允许所有 ICMP（更宽松）：
iptables -A INPUT -p icmp -j ACCEPT

# 3. 允许本地回环接口 (lo) 的所有流量
iptables -A INPUT -i lo -j ACCEPT

# 4. 允许新的 TCP 连接访问 SSH（端口 22）
iptables -A INPUT -p tcp --dport 22 -m conntrack --ctstate NEW -j ACCEPT

# 5. 添加兜底策略
iptables -A INPUT -j REJECT --reject-with icmp-host-prohibited
```

1. 默认展示的是 filter 表的链，包含三条链：INPUT、OUTPUT、FORWARD
2. INPUT 下包含 13 条规则（省略了 5 到 12 条的数据）

4. 第二条为接收 icmp 协议的包。
5. 第三条为发送到本地回环网卡 lo 的包全部接受。
6. 第四条为建立 tcp 连接且端口号为 22 的包全部接受。
7. 第十三条为拒绝全部的包。

具体情况

1. ssh 登录

第一条不满足；第二条不满足；第三条不满足；第四条建立 tcp 22 端口连接满足放行（结束）。建立连接之后后续的包都满足第一条全部放行。

1. 比如访问 23 端口

第一条不满足；------ 满足第十三条，第十三条为丢弃（结束）

设置默认的策略

上述有 13 条规则，如果进来的包都不匹配全部的十三条规则的话（上述的案例中不会发生这种情况，因为第13条规则匹配所有的包进行拒绝），那么我们可以设置默认的策略进行兜底。

```
iptables -P INPUT DROP

iptables -P OUTPUT ACCEPT

iptables -P FORWARD ACCEPT
```


安装服务
```
apt install iptables-persistent
```

手动保存、恢复规则
```
netfilter-persistent save
iptables-restore < /etc/iptables/rules.v4
```

`iptables-persistent` 它会在每次系统启动时 ​**​自动恢复你之前保存的规则​**​，无需手动操作。但是保存需要手动保存。

## nftables
查看所有的表格
```
# nft list tables
table inet filter
```

输出格式是 `table [IP格式] [表格名称]`

IP 格式：  
- ip：仅针对 IPv4 的包做分析。
- ip6：仅针对 IPv6 的包做分析。
- inet：同时针对 IPv4 和 IPv6 的包做分析。
- arp：主要针对 IPv4 的 ARP 包做分析。
- bridge：分析来自桥接器装置的包。
- netdev：分析来自某个网络卡的包。

表是第一层容器，创建时仅需要指定协议和名称，是纯粹的命名空间。

添加表格
```
# nft add table ip mytable
```

列出所有表格内的链
```
# nft list chains
table inet filter {
        chain input {
                type filter hook input priority filter; policy accept;
        }
        chain forward {
                type filter hook forward priority filter; policy accept;
        }
        chain output {
                type filter hook output priority filter; policy accept;
        }
}
```

链位于表内，是规则的容器。规则是最终的执行单元。同 hook 阶段，不同表的链也是按优先级顺序执行，一旦有链匹配了 `accept/drop/reject`，包就结束处理，后续链不会再执行。

链的定义
```
type [链的类型] hook [针对包的流向] priority [优先级]; policy [执行动作];
```
type 决定了使用哪个 netfilter 子系统：
- filter：可以挂在所有的 hook 阶段。
- nat：只能挂在 prerouting、postrouting、input、output 阶段。
- route：只能挂在 output 阶段。

hook 表示在网络栈的哪个阶段执行，取值范围：
- ingress：包进入网络接口之后，协议栈之前。
- prerouting：路由决策之前。
- input：路由决策之后，发往本机进程之前。
- forward：路由角决策之后，转发。
- output：本机进程处理完成之后。
- postrouting：最终处理，网络接口发送出去之前。

入站流量路径（从外部网络到本机）：
外部网络 → ingress (netdev) → prerouting → 路由决策 → input → 本机进程

转发流量路径（经过本机转发）：
外部网络 → ingress (netdev) → prerouting → 路由决策 → forward → postrouting → 外部网络

出站流量路径（从本机到外部网络）：
本机进程 → output → postrouting → 外部网络

priority 表示优先级，数值越小优先级越高。如果设置的是文本，其对应的优先级数值如下：
- raw -300
- mangle -150
- dsnat -100(ip,ip6,inet)/-300(bridge)
- filter 0(ip,ip6,inet,arp,netdev)/-100(bridge)
- security 50
- srcnat 100(ip,ip6,inet)/300(bridge)
- out 100


添加链
```
nft 'add chain ip mytable myinput { type filter hook input priority filter; policy accept; }'
```

查看链
```
nft list chain ip mytable myinput
```

nftables 从上到下匹配规则，匹配到第一个符合条件的规则就执行动作，不再往下匹配。
规则的语法
```
<match> <action> [# comment]
```

常见的匹配条件
```
ip saddr - 源IP，示例：ip saddr 192.168.31.122
ip daddr - 目的 IP，示例：ip daddr 10.0.0.0/24
tcp dport - TCP 目的端口，示例：tcp dport 22
udp sport - UDP 源端口，示例：udp sport 53
iifname - 入接口，示例：iifname "eth0"
oifname - 出接口，示例：oifname "eth1"
meta l4proto - 协议类型，示例：meta l4proto icmp
ct state - 连接追踪状态，示例：ct state established,related
```

常见的动作
```
accept 放行
drop 丢弃，不返回任何信息
reject 拒绝匹配的包，并返回 ICMP/TCP 拒绝消息
log 打印日志
counter 增加计数器
jump <chain> 跳转到另一条链继续匹配
return 返回到上一条链继续匹配
```

添加规则
```
nft add rule [IP类型] [表] [链] [规则项目]
```

清空所有规则
```
nft flush ruleset
```

添加规则实际操作
```
nft add rule ip mytable myinput ct state established,related accept
nft add rule ip mytable myinput meta l4proto icmp accept
nft add rule ip mytable myinput iifname lo accept
```

查看规则
```
# 列出 ip 类型的规则，并且列出 handle 点（-a）
# nft -a list ruleset ip
table ip mytable { # handle 2
        chain myinput { # handle 1
                type filter hook input priority filter; policy accept;
                ct state established,related accept # handle 4
                meta l4proto icmp accept # handle 5
                iifname "lo" accept # handle 6
        }
}
```

插入规则
```
# nft insert rule ip mytable myinput handle 5 ip saddr 10.30.30.120 tcp dport 22 accept
# nft -a list ruleset ip
table ip mytable { # handle 2
        chain myinput { # handle 1
                type filter hook input priority filter; policy accept;
                ct state established,related accept # handle 4
                ip saddr 10.30.30.120 tcp dport 22 accept # handle 7
                meta l4proto icmp accept # handle 5
                iifname "lo" accept # handle 6
        }
}
```

替换/删除规则
```
使用 nft replace rule [IP類型] [table] [chain] handle [point] 規則
使用 nft delete rule [IP類型] [table] [chain] handle [point]
```

nft add 添加规则之后会立即生效，但是重启服务不会保留。将规则保存到配置文件，Debian 默认开机加载 `/etc/nftables.conf`。
```
# nft list ruleset > /etc/nftables.conf
```

实践
```
nft insert rule inet filter input ct state invalid drop
nft add rule inet filter input iif lo accept
nft add rule inet filter input meta l4proto icmp accept
nft add rule inet filter input tcp dport 22 accept
nft add rule inet filter input drop
```
